# Red Hat en desarrollo y DevOps

Red Hat ofrece sistemas operativos, middleware y herramientas orientados a entornos empresariales y cloud. En desarrollo y DevOps destacan RHEL como base estable y OpenShift como plataforma de contenedores y orquestación.

## Componentes principales

| Producto / concepto                 | Descripción                                      | Uso típico                                    |
| ----------------------------------- | ------------------------------------------------ | --------------------------------------------- |
| **RHEL** (Red Hat Enterprise Linux) | Distribución Linux empresarial con soporte largo | Servidores, imágenes base para contenedores   |
| **OpenShift**                       | Plataforma Kubernetes empresarial (PaaS)         | CI/CD, despliegues, escalado, gobernanza      |
| **OpenShift Pipelines**             | Tekton en OpenShift para pipelines CI/CD         | Build, test y deploy como código              |
| **Quay**                            | Registro de contenedores                         | Almacenamiento y escaneo de imágenes          |
| **Ansible**                         | Automatización (ahora parte de Red Hat)          | Configuración, aprovisionamiento, despliegues |

## Por qué Red Hat en DevOps

- **Estabilidad y soporte:** RHEL y OpenShift tienen ciclos de vida predecibles y parches de seguridad.
- **Kubernetes gestionado:** OpenShift añade autenticación, monitoreo, registro y flujos de desarrollo sobre Kubernetes.
- **Ecosistema integrado:** Integración con JBoss, Red Hat Fuse, Ansible y servicios cloud (AWS, Azure, GCP) mediante operadores y herramientas Red Hat.

En resumen, Red Hat se elige cuando el contexto es empresarial, se requiere soporte y gobernanza, y se quiere una plataforma Kubernetes lista para producción.
