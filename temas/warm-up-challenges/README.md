# Warm-up Challenges

New to HackerRank? You can try these easy ones first.

## Qué son

Desafíos sencillos para familiarizarse con la plataforma y con el flujo de leer enunciado, escribir código en el editor y pasar tests. Suelen implicar operaciones básicas: lectura de entrada, impresión de salida, condicionales y bucles simples.

## Conceptos clave

- Entrada/salida estándar (stdin/stdout).
- Tipos primitivos y operaciones básicas (aritmética, comparación).
- Condicionales (`if`, `else`) y bucles (`for`, `while`).
- Formato de salida (espacios, saltos de línea).

## Datos clave

- Nivel: fácil.
- Objetivo: ganar confianza antes de problemas con estructuras de datos o algoritmos más avanzados.
- Tiempo típico: pocos minutos por problema.

## Ejemplo de problema típico

Dado dos enteros `a` y `b`, imprimir su suma.

**Entrada:** dos líneas con un entero cada una.  
**Salida:** un entero, la suma.

**Ejemplo (pseudocódigo):**

```
leer a
leer b
imprimir a + b
```

**Ejemplo (Java):**

```java
Scanner sc = new Scanner(System.in);
int a = sc.nextInt();
int b = sc.nextInt();
System.out.println(a + b);
```

Otros patrones frecuentes en warm-up: comparar dos números, imprimir el mayor o menor, o repetir una acción un número fijo de veces.
